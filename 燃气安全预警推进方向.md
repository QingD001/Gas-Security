# 燃气安全预警推进方向

---

## 一、现状回顾

当前系统基于**专家规则引擎**，对可燃气体探测仪的预警数据进行真假判定。通过对683条已标注历史数据（误报337条，真实预警346条）的深度挖掘，构建了5条高精确率误报拦截规则。

| 指标 | 当前值 | 说明 |
|------|--------|------|
| 精确率 | 95.8% | 命中规则的报警基本可确认为误报 |
| 召回率 | 20.5% | 仍有约80%复杂误报无法被规则覆盖 |
| 规则数量 | 5条 | R1极小值/R2阈值异常/R3突变/R4孤立/R6持续极小值 |

**核心局限性**：
- 简单规则无法覆盖"形似真实"的复杂误报模式（如渐变上升后回落）
- 规则引擎是静态的，无法从新数据中自学习
- 当前规则和判断均依赖**报警后数据**，属于事后判定

---

## 二、整体推进方向：规则引擎 + XGBoost 融合架构

采用**串联漏斗架构**，兼顾确定性拦截与智能识别：

```
报警数据流
    │
    ▼
┌─────────────────────────┐
│ 第一层：专家规则引擎      │  → 拦截高置信度硬件故障/配置错误
│ （保留现有5条规则）       │     精确率 > 95%，毫秒级判定
└──────────┬──────────────┘
           │ 规则未命中
           ▼
┌─────────────────────────┐
│ 第二层：XGBoost 模型     │  → 输出 0~1 误报概率
│ （全量时序特征）          │     突破规则覆盖率天花板
└──────────┬──────────────┘
           │
           ▼
    概率阈值动态调优
    ├── 保守策略：误报概率 > 0.9 → 自动消警
    └── 激进策略：误报概率 > 0.7 → 自动消警
```

**核心思路**：第一层快速拦截确定性误报，第二层用XGBoost学习复杂时序模式，输出可调节的概率值，业务端可根据容忍度灵活配置阈值。

## 三、问题一：传输间隔差异 — ML/DL 层适配方案

### 3.1 问题分析

不同设备的推送间隔不完全一致（部分间隔5分钟，部分30分钟甚至更长）。同一模型如果直接吃入不同间隔的原始序列，提取出的统计特征（如"连续5次为0"、"序列方差"）在物理上不可比——5次一跳对快设备是25分钟、对慢设备是150分钟。

ML/DL 层有两种思路解决这个问题，核心差异在于：**要不要把数据预处理成统一规格**。

### 3.2 路线一：数据预处理 → 统一时间网格（重采样）

将不同间隔的原始序列**全部重采样到统一时间网格**（如1分钟1个点），缺失位置用前值填充（forward fill）或线性插值，得到等间隔、等长度的序列，再喂入模型。

```
设备A（5min间隔，原始11点）   → 统一1min重采样 → 统一150点序列
设备B（30min间隔，原始11点）  → 统一1min重采样 → 统一150点序列
                                      ↓
                       XGBoost / LSTM 输入维度完全一致
```

**优势**：
- 模型和特征工程完全不用感知底层采样差异，架构简洁
- 趋势斜率、方差、变化速率等统计量在统一网格上计算才有真正的可比性
- 深度学习模型（LSTM/Transformer）天然要求等长输入，重采样可直接适配

**代价**：
- 引入插值噪声。极端情况下30min间隔设备只有11个原始点，却要插出上百个点——信息量并未增加，但给了模型"数据很密"的假象
- 重采样后序列变长，计算开销增大

### 3.3 路线二：时间感知模型（保留原始不等间隔）

保留原始的不等间隔序列，让模型**显式感知时间间隔信息**，从数据本身学习采样率差异带来的影响。

**XGBoost 路径**：

将时间间隔信息作为附加特征，与数值统计特征一同输入：

| 特征类别 | 示例 |
|----------|------|
| 间隔统计量 | 相邻推送时间差的均值、标准差、最大/最小间隔 |
| 时间加权统计 | 以时间间隔为权重，重新计算序列均值、方差（间隔大的点权重大） |
| 变化速率 | (相邻值之差) / (时间差)，即浓度变化率而非绝对变化量 |
| 距报警时间跨度 | 第一条推送距报警时刻的总时长（反映设备采样能力） |

本质是将"时间信息"显式编码为特征，让XGBoost自动学会不同采样率下的误报规律。

**LSTM/Transformer 路径**：

将每次推送的**时间差 Δt**（距上一次推送的间隔）作为额外输入维度，与浓度值拼接输入：

```
时刻:    t₁          t₂          t₃     ...   t_alarm
浓度:   [0.01]      [0.02]      [0.15]  ...   [0.38]
Δt:     [ — ]      [ 5min ]    [ 5min ] ...   [ 5min ]
         ↓           ↓           ↓               ↓
    输入向量 = concat(浓度, Δt编码)
```

或者使用**时间感知的位置编码**（Time-Aware Positional Encoding），让 Transformer 的自注意力机制直接感知时间距离，而非仅依赖序列位置。

**优势**：
- 不引入插值噪声，信息保真度高
- 模型能学到"间隔大时取值波动理应更大"这类物理规律
- XGBoost 路径实现成本较低

**代价**：
- XGBoost 需要手工设计时间相关特征，依赖领域知识
- LSTM/Transformer 路径：不等长序列需要 padding/masking，模型设计和训练更复杂
- 长度差异过大的序列（如一个10点、一个200点）仍需额外处理

### 3.4 路线对比

| 维度 | 路线一：重采样统一网格 | 路线二：时间感知模型 |
|------|----------------------|---------------------|
| 实现复杂度 | 低（预处理为主） | 中（XGBoost）/ 高（LSTM） |
| 信息保真度 | 中（插值引入噪声） | 高 |
| 模型适配 | 所有模型均可 | XGBoost 易、深度学习需额外设计 |
| 计算开销 | 偏高（序列变长） | 低（保持原始长度） |
| 对极稀疏设备的鲁棒性 | 差（插值太多） | 较好（显式标记间隔） |
| 可解释性 | 好 | 好（XGBoost）/ 中（深度学习） |

> 建议：两条路线在初期均可小规模验证，对比在同一标注集上的模型表现后再选定主路线。

## 四、问题二：报警时刻实时误报概率判定

### 4.1 问题分析

当前规则和时序模型的判定逻辑**依赖报警后数据**（如R4需看报警后5次推送是否全零，R6需看报警后极小值比例），属于**事后判定**模式。业务上需要在报警产生时立即给出误报概率，仅使用**报警前数据 + 报警当前值**。

核心挑战：数据量减半（从21点减至11点），缺少报警后的趋势信息。

### 4.2 解决方案：双模型分阶段判定

```
报警发生时刻                          报警后10次推送收集完毕
    ├──────────────────────────────────────┤
    │                                      │
    ▼                                      ▼
┌──────────────────┐              ┌──────────────────┐
│ 实时模型（阶段一） │              │ 事后模型（阶段二） │
│ 输入：报警前10次  │     →→→     │ 输入：全序列21点  │
│ + 报警当前值     │   复核确认    │ （前后各10次）    │
│ 输出：快速误报概率 │              │ 输出：最终误报概率 │
└──────────────────┘              └──────────────────┘
   用途：即时分级处置                 用途：最终判定归档
```

**实时模型（阶段一）特征方向**：

仅使用报警前10次推送+报警时刻的数据，核心提取以下维度的特征：

| 特征维度 | 说明 | 对误报判定的意义 |
|----------|------|-----------------|
| 报警前趋势斜率 | 线性拟合报警前序列的斜率 | 真实泄漏通常有渐变上升趋势 |
| 数值稳定性 | 报警前序列的方差/变异系数 | 传感器故障常表现为剧烈震荡 |
| 报警值/历史峰值比 | 当前报警值与报警前最大值的比值 | 瞬时跳变多为干扰信号 |
| 报警前零值比例 | 报警前序列中零值或极小值的占比 | 前序全零突变多为毛刺干扰 |
| 距上次非零时间 | 报警时刻距最后一次有效读数的时间 | 长时间静默后突发的可信度低 |
| 报警值/阈值比 | 超出阈值的幅度 | 略超阈值 vs 远超阈值的行为模式不同 |

**双模型协作策略**：

- 实时模型侧重"宁可漏判、不可误判真实报警"，采用较保守的阈值，仅拦截高置信度误报
- 报警后数据收集完成后，事后模型以更丰富的信息进行复核，给出最终判定
- 两个模型可输出一致的0~1概率值，业务系统无需感知模型切换

**可解释性保证**：XGBoost天然支持特征重要性排序，对每个判定可输出"Top 3 影响特征"，业务人员可理解判断依据（如"报警前序列方差过高 + 报警值/峰值比过大"）。

### 4.3 预期效果

实时模型在仅用一半数据的前提下，仍能有效拦截明显误报模式（如报警前全零突变、持续震荡等），预期可实现：
- 报警时刻即时拦截率：可覆盖专家规则中不依赖报警后数据的误报类型
- 事后模型完整判定拦截率：60%~80%

## 六、收益预期总结

| 维度 | 当前专家规则 | 推进后（规则 + XGBoost） |
|------|-------------|------------------------|
| 误报拦截率 | ~20% | 实时 > 50% / 事后 60%~80% |
| 判定时机 | 事后（依赖报警后数据） | 报警时刻即时 + 事后复核 |
| 设备适配 | 统一阈值，不区分间隔 | 时间窗口 + 间隔特征自适应 |
| 可解释性 | 规则命中即给出明确原因 | 特征重要性 + Top-N 归因 |
| 进化能力 | 人工复盘调整 | 新数据持续训练迭代 |
| 安全兜底 | 精确率 95.8% | 保留规则引擎，精确率不降级 |
