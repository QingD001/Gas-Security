# 燃气安全预警模型阶段汇总

时间：2026-05-22

这次主要把任务口径和实验产物重新捋了一遍。现在这版不是做“未来会不会触发阈值”，而是看一段 20 次监测流水，判断它像不像一次真报警。

模型输出是一个概率分数：

```text
true_alarm_prob 越高，说明这段流水越像真报警。
```

后续计算精确率、召回率、F1时以0.5为阈值：

```text
true_alarm_prob >= 0.5  判为 true_alarm
true_alarm_prob < 0.5   判为 non_true_alarm
```

## 数据集

当前数据集文件：

```text
mid_dataset.csv
```

构建脚本：

```text
scripts/remake_data.py
```

这版数据集用了 `data.csv` 里全部明确标注的报警样本：

```text
非误报 390 条 -> true_alarm
误报   425 条 -> false_alarm
```

没有再因为 20 次流水缺值多就把样本丢掉。缺值、零值、有效值数量本身可能就是误报特征，所以现在保留下来，交给特征工程处理。

同时从大流水分片中抽取普通不报警窗口：

```text
no_alarm 2445 条
```

最终数据量：

```text
no_alarm       2445
false_alarm     425
true_alarm      390
总计           3260
```

## 模型方法

当前模型代码：

```text
ML2.py
```

模型使用 XGBoost。训练目标是：

```text
true_alarm = 1
false_alarm + no_alarm = 0
```

也就是把“真报警”作为高风险样本，把“误报”和“普通不报警”都作为低风险样本。

特征工程函数在 `ML2.py` 里：

```python
feature_engineering(df)
```

当前没有把阈值相关特征直接喂给模型。模型主要看 20 次流水本身和窗口形态。

主要特征包括：

```text
v_1 ~ v_20
窗口均值、最大值、最小值、标准差
前中后三段统计
上升比例、下降比例
最大值位置
连续零值
有效值数量
偏度、峰度
监测指标类型
```

## 当前结果

本轮训练测试按 7:3 随机划分：

```text
训练集 2282 条
测试集 978 条
```

测试结果：

```text
TP=73, FP=30, TN=831, FN=44
精确率=70.87%
召回率=62.39%
F1=66.36%
AUC=0.9615
```

各类样本的平均预测分数：

```text
true_alarm   0.579
false_alarm  0.225
no_alarm     0.021
```

这个结果说明：模型能比较明显地把普通不报警窗口压低，也能把真报警整体抬高。误报样本的平均分比真报警低，但仍有一部分误报被打成高风险，这部分需要后面结合业务再看。

测试集中预测错误的样本：

```text
true_alarm  漏判 44 条
false_alarm 误判 25 条
no_alarm    误判 5 条
```

这里最值得分析的是前两类：

```text
真报警为什么被压低；
误报为什么被模型认为像真报警。
```
